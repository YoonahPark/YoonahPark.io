함수 (Function)

 

  • 함수는 특정한 단일 기능을 가지는 코드의 집합이다. 
  • 일반적으로 함수를 선언할 때는 괄호 ( )내부에 매개변수(Parameter)라는 변수를 선언하게 되며, 함수를 호출할 때는 괄호 ( ) 내부에 인자(Argument)라는 값을 입력하게 된다.
  • 파이썬 함수는 자유롭게 입력값(매개변수)를 입력받거나 입력받지 않을 수 있으며, 출력값(return 값)을 출력하거나 출력하지 않을 수 있다. 
  • 함수에는 내장함수와 외장함수가 있는데, 외장함수는 import한 외부 모듈 내에 있는 함수이며, 내장함수는 import할 필요가 없는 Built-in function이다.
함수 : 특정한 단일 기능을 가지는 코드의 집합

 

클래스 (Class)

 

  • C와 같은 언어를 '절차 지향 프로그래밍 언어'라고 부르고, python, Java, C++ 등의 언어는 객체 지향 프로그래밍을 지원하므로 '객체 지향 프로그래밍 언어'라고 부른다. 객체 지향 프로그래밍(Object Oriented Programming)은 클래스(class)를 기반으로 객체(object)를 만들고 객체를 중심으로 프로그래밍하는 것이다. 
  • 클래스는 객체를 만들 수 있도록 하는 틀이며, 클래스를 기반으로 만들어진 객체의 실제 구현물을 인스턴스라고 한다.
클래스 : 객체를 만들 수 있도록 하는 틀 

 

메소드 (Method)

 

  • 메소드는 클래스, 구조체, 열거형에 포함되어 있는 함수를 일컫는다. 
메소드 : 클래스, 구조체, 열거형에 포함되어 있는 함수
클래스 : 객체를 만들기 위한 틀
객체 : 클래스 형태를 기반으로 만들어지는 물체
인스턴스 : 클래스 형태를 기반으로 만들어지는 물체의 실제 구현물
클래스/인스턴스 함수 : 클래스 내에 포함된 함수, 클래스 내에서의 동작(action)을 구현 (=메소드) 
클래스/인스턴스 변수 : 클래스 또는 인스턴스의 속성(attribute)을 나타냄

 

모듈 (Module)

 

    일반적으로 이야기하는 '모듈'은 전체가 보다 작고 이해할 수 있는 단위로 나뉘어진 것을 이야기한다. 레고 작품을 구성하는 각각의 블록처럼, 모듈은 한 가지 기능을 수행하며 단순하고 독립적인 작은 단위다. 이러한 모듈 여럿을 조립하여 전체를 완성할 수 있다. 비슷한 맥락에서 문제를 '모듈화'한다는 것은 커다란 문제를 작은 문제 조각로 분할하여 이해하고 처리하기 쉽도록 만드는 것을 의미한다. 

    컴퓨터 프로그래밍에서 '모듈'의 의미 역시 이와 유사하다. 모듈은 완결된 구조를 통해 특정한 단일 기능을 수행하며 서로 독립적인 함수, 변수, 클래스 등의 모음을 의미한다. 모듈은 프로그램의 일부를 구성할 수 있는 유용한 함수, 자료들을 모아놓은 소프트웨어 묶음이며, 필요할 때 마다 가져다 쓸 수 있도록 한다. 

    파이썬의 경우에는 특정 기능을 .py 단위로 작성했을 때 모듈이라고 부른다. 스크립트(Script) 역시 같은 뜻으로 사용된다.

모듈은

1. 특정한 단일 기능/목적을 가짐
2. 그 자체로 완결된 구조를 가진 실체
3. 각각이 독립적임
4. 프로그램의 일부를 구성하는 것에 유용성이 있음

위와 같은 특성을 가지는 '함수, 변수, 클래스 등이 모여 특정 기능을 구현하는 파일'이다.

파이썬의 경우에는 특정 기능을 .py 단위로 작성했을 때 모듈이라고 부른다.

 

패키지 (Package)

 

  • 패키지는 특정 기능과 관련된 여러 모듈들을 하나의 상위 폴더에 넣어 놓은 것이다.
  • 패키지 내에 하위 폴더가 더 존재할 수 있다.
  • 파이썬 3.3 이전에는 폴더 내에 __init__.py 파일을 생성해야 패키지로 인식하였다.
  • 외부 패키지는 외부 라이브러리에 있는 패키지를 일컫는다.
  • A.B라고 하면 A는 패키지, B는 모듈이 됩니다. (예: sound.echo –> sound 패키지에 있는 echo모듈)
패키지 : 특정 기능과 관련된 여러 모듈들을 하나의 상위 폴더에 넣어 놓은 것

 

라이브러리 (Library)

 

  • 일반적으로 여러 모듈과 패키지를 묶어 라이브러리라고 한다.
  • 파이썬에는 python.org에서 개발한 표준 라이브러리(Python Standard Library)와 3rd party에서 개발한 외부 라이브러리(Third Party Library)가 있다.
  • 표준 라이브러리에는 흔히 아는 time, sys, os, math, random, urllib 등이 있다.
  • Numpy, Scipy, requests, scrapy, webbrowser 등은 외부 라이브러리다.
라이브러리 : 여러 모듈과 패키지, 내장 함수를 묶은 것

 

단, 프로그래밍 언어에 따라 패키지와 라이브러리의 정의는 달라질 수 있다. 파이썬에서는 패키지와 라이브러리를 같은 의미로 사용하기도 한다.
